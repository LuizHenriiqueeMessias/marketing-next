# Phase 3 — UI Design Contract

> Visual and interaction contract for the Ad Intelligence Dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS variable system |
| Preset | not applicable |
| Component library | Hand-rolled (src/components/ui/) using CSS variables from Design-System-Criativos.html |
| Icon library | lucide-react (already in use) |
| Font | Inter (sans-serif) — `--cr-font: 'Inter', sans-serif` |

**Source:** Design-System-Criativos.html `:root` block + existing components (button.tsx, badge.tsx).

No `components.json` exists. The project uses CSS custom properties (`var(--bg)`, `var(--surface)`, `var(--accent)`, etc.) applied globally. All new components must use these same variable names, not hardcoded hex values.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact row spacing, gap between filter chips |
| md | 16px | Default element spacing, card internal padding base |
| lg | 24px | Card padding, filter bar internal padding |
| xl | 32px | Section vertical gaps, page content padding-top |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical breathing room |

Exceptions:
- Touch targets for icon-only action buttons: minimum 32px × 32px (matches existing `h-7 w-7` ghost button pattern)
- Ad card thumbnail height: 160px (fixed, not scale-derived — aspect ratio constraint for creative preview)
- Tab bar pill height: 36px (matches existing Inspiracao tab pattern: `py-2` + `rounded-lg`)
- Detail page column scroll area: height = `calc(100vh - 120px)` (sticky header offset)

**Source:** Design-System-Criativos.html spacing section + existing Inspiracao index.tsx patterns.

## Typography

| Role | Size | Weight | Line Height | CSS Variable | Usage |
|------|------|--------|-------------|--------------|-------|
| Body | 13px | 400 (regular) | 1.6 | `var(--cr-font)` | Table cell text, card body, filter labels |
| Label | 12px | 600 (semibold) | 1.4 | — | Form field labels, badge text, column headers |
| Heading | 16px | 600 (semibold) | 1.3 | — | Page section headers, card competitor name, dialog titles |
| Display | 24px | 600 (semibold) | 1.2 | — | Page title "Ad Intelligence" only |

**Weights declared: 2** — 400 (regular) for body prose, 600 (semibold) for all emphasis, headings, labels, and display. Weight 700 is not used.

Font stack: `'Inter', sans-serif` for all body/label/heading. Page header title uses Montserrat 600 to match existing `page-header-title` class pattern (source: Inspiracao/index.tsx line 608). Note: if the existing `page-header-title` class renders at bold/700 via the stylesheet, the class itself is reused as-is — no new weight token is introduced.

Monospace (score values, IDs): `'Outfit', monospace` — used for numeric score display (source: Design-System-Criativos.html `.color-value`).

**Body 13px / Label 12px proximity note:** These two sizes are 1px apart. Differentiation is achieved by color: Body uses `var(--text-1)` / `var(--text-2)` (higher contrast), Label uses `var(--text-3)` (muted, reduced contrast). Size alone is not the sole differentiator — color pairing makes the hierarchy legible.

**Source:** Design-System-Criativos.html typography section + body `line-height: 1.6` + existing component patterns.

## Color

| Role | Value | CSS Variable | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | `#0a0a0a` | `var(--bg)` / `var(--cr-bg)` | Page background, content area |
| Secondary (30%) | `rgba(255,255,255,0.03)` | `var(--surface)` / `var(--cr-surface)` | Cards, table rows, tab bar, sidebar, dialog backgrounds (`#141414`) |
| Accent (10%) | `#e8604a` | `var(--accent)` / `var(--cr-accent)` | See reserved list below |
| Destructive | `#e85454` | `var(--cr-red)` | Delete competitor button, confirm delete actions only |

**Accent reserved for:**
- Primary CTA buttons (gradient `var(--cr-grad)`: `linear-gradient(135deg, #c2396e 0%, #e8604a 100%)`)
- "Coletar Anúncios" trigger button (primary variant)
- "Adicionar Concorrente" button (primary variant)
- Active tab indicator background
- ScoreBar `filled-high` dots (>= 8: `#4ade80`), `filled-mid` (>= 6: `#fbbf24`), `filled-low` (< 6: `#ef4444`)
- Score numeric label color classes: `.high`, `.mid`, `.low`
- Active nav item in sidebar
- Border highlight on focused/hovered interactive elements (`var(--cr-border-hover)`)

**Score semantic colors (separate from accent):**
- High score (>= 8): `#4ade80` — `var(--cr-score-high)`
- Mid score (6–7): `#fbbf24` — `var(--cr-score-mid)`
- Low score (< 6): `#ef4444` — `var(--cr-score-low)`

**Format badge colors (existing system, reused verbatim):**
- Video: bg `rgba(168,85,247,0.10)`, text `#a855f7`, border `rgba(168,85,247,0.25)` — `badge-video`
- Carousel: bg `rgba(59,130,246,0.10)`, text `#3b82f6`, border `rgba(59,130,246,0.25)` — `badge-carousel`
- Image: bg `rgba(34,197,94,0.10)`, text `#22c55e`, border `rgba(34,197,94,0.25)` — `badge-image`

**Status badge colors:**
- Active (`ativo`): `badge-status-done` — bg `var(--cr-green-muted)`, text `var(--cr-score-high)`
- Inactive (`inativo`): `badge-status-pending` — bg `var(--cr-amber-muted)`, text `var(--cr-score-mid)`
- needs_reanalysis: `badge-status-error` — bg `var(--cr-red-muted)`, text `var(--cr-red)`

**Source:** Design-System-Criativos.html `:root` CSS variables + badge.tsx variant definitions.

## Interaction Contracts

### Filter Bar (Ads tab)

All filters are client-side state. Data loaded once on mount via:
```
supabase.from('ad_creatives')
  .select('*, ad_competitors(*), ad_analyses(*)')
  .order('collected_at', { ascending: false })
```

Filter state shape:
```typescript
{
  grupo: string | null,         // matches ad_competitors.grupo
  competitorId: string | null,  // matches ad_creatives.competitor_id
  format: 'video' | 'image' | 'carousel' | null,  // creative_type
  minScore: number | null,      // ad_analyses.score >= minScore
  startDateFrom: string | null, // ad_creatives.start_date >= value
  startDateTo: string | null,   // ad_creatives.start_date <= value
  status: 'ativo' | 'inativo' | null,  // ad_creatives.status
}
```

Filter controls: all use `<select>` with `.select` CSS class pattern (height 40px, surface bg). Score minimum: options `[Qualquer, 7+, 8+, 9+]`. Date inputs: `<input type="date">` styled as `.input`.

Active filter count badge: shown on "Filtros" label when any non-null filter is active — `badge-accent` variant.

### View Mode Toggle

Two buttons side by side: `[⊞ Cards]` `[≡ Tabela]`. Active state: `background: var(--cr-grad)`, inactive: outline style. State persisted to `localStorage` key `'ad-intel-view'`.

### CSV Export

Triggered by "Exportar CSV" button. Operates on the currently filtered `ad_creatives` array in memory. No API call. Fields exported:
`id, competitor_name, grupo, creative_type, status, start_date, end_date, body_text, cta_type, score, hook_text, hook_type, angle_tag, cta_analysis, structure_summary, insights, collected_at`

Button: `variant="outline"`, size `sm`, icon `Download` (lucide), label "Exportar CSV". Positioned at far right of filter bar.

### Collection Trigger (Competitors tab)

Per-row "Coletar" button: `variant="outline"` size `sm`, icon `Play` (lucide). On click: calls `POST /api/ad-intelligence/collect` with `competitor_id`. Button shows `Loader2` spinning during call (replaces Play icon). On success: `toast.success('Coleta iniciada!')`. On error: `toast.error(message)`. Button re-enables after response (fire-and-forget — backend processes async).

### Add Competitor Dialog

Fields: Nome (required), Page ID (optional), Page URL (optional), Grupo (text, optional), Notas (textarea, optional).
Submit button: primary variant, label "Adicionar Concorrente".
Validation: Name field required — inline error "Nome é obrigatório" below field in `var(--cr-red)` at 12px.

### Delete Competitor

Uses `AlertDialog` with:
- Title: "Remover concorrente?"
- Description: "Todos os anúncios coletados de [nome] serão removidos permanentemente."
- Cancel: outline button
- Confirm: destructive button "Remover"

Cascade delete is handled by `ON DELETE CASCADE` in schema — no orphan cleanup needed in frontend.

### Animations

All list mounting: `framer-motion` `motion.div` with `initial={{ opacity: 0, y: 8 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.2 }}`. Matches existing PostsTable.tsx pattern.
Card hover: CSS `transition: border-color 0.2s, transform 0.15s` — no JS animation.
Tab switch: no animation (immediate content swap, consistent with existing Inspiracao tabs).

## States and Fallbacks

| State | Visual Treatment |
|-------|-----------------|
| Loading (initial data fetch) | `Loader2 className="w-4 h-4 animate-spin"` centered in `py-16` container, `color: var(--text-3)` |
| Loading skeleton (card grid) | 6 skeleton cards — `div` with same card dimensions, `background: var(--surface)`, `border: 1px solid var(--border)`, `animate-pulse` Tailwind class |
| No thumbnail (`storage_image_path` null) | Gray placeholder: `div` 160px height, `background: var(--surface)`, centered `Image` icon 24px at `var(--text-3)` |
| No video (`storage_video_path` null for video type) | Same gray placeholder with `Video` icon |
| Analysis row `null` / missing | Em dash "—" in `var(--text-3)` — never blank, never "undefined" |
| `needs_reanalysis = true` | `badge-status-error` "Análise incompleta" in detail page AI Analysis header |
| Competitor `is_active = false` | Row shown with reduced opacity (`opacity: 0.5`) in competitors table; "Coletar" button disabled |
| Filter active with 0 results | Empty state variant 3 (see Copywriting Contract) |
| Detail page — ad not found | Full-page centered: "Anúncio não encontrado" + back button |
