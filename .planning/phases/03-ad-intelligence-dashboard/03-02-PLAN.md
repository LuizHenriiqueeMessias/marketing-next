    **CompetitorList.tsx** — Follow ProfileList.tsx pattern (per D-13, D-14, D-15):

    1. State: competitors[] (AdCompetitor[]), loading, search, dialogOpen (add), editTarget, deleteTarget, collectingId (string|null)
    2. fetchCompetitors: `supabase.from('ad_competitors').select('*').order('created_at', { ascending: false })`
    3. useEffect on mount to fetch
    4. Header row: Button "Adicionar Concorrente" (primary variant, Plus icon) + search Input
    5. Table with columns per UI-SPEC: Nome | Page ID | Grupo | Status (Switch) | Coletar | Acoes (edit Pencil + delete Trash2)
    6. **Switch toggle** for is_active: onChange calls `supabase.from('ad_competitors').update({ is_active: !current }).eq('id', id)`, then refetch. Disabled rows (`is_active=false`) have `opacity: 0.5` on the row.
    7. **Coletar button** per row: `variant="outline"` size `sm`, shows Play icon normally. On click:
       - Set collectingId to competitor.id
       - `fetch(\`${import.meta.env.VITE_API_URL}/ad-intelligence/collect\`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ competitor_id: competitor.id }) })`
       - On success: `toast.success('Coleta iniciada! Os anuncios aparecerao em alguns minutos.')`
       - On error: `toast.error(\`Erro ao iniciar coleta: ${err.message}\`)`
       - Finally: set collectingId to null
       - Button disabled when `is_active=false` or `collectingId === competitor.id`
       - When collectingId matches, show Loader2 with animate-spin instead of Play icon
    8. **Delete** uses AlertDialog per UI-SPEC destructive pattern:
       - Title: "Remover concorrente?"
       - Description: "Todos os anuncios coletados de {name} serao removidos permanentemente. Esta acao nao pode ser desfeita."
       - Confirm: destructive button "Remover"
       - Action: `supabase.from('ad_competitors').delete().eq('id', id)` then refetch
    9. **Empty state**: when competitors.length === 0 and not loading, show centered div with Inbox icon, heading "Nenhum concorrente cadastrado", body "Adicione o primeiro concorrente para comecar o monitoramento."
    10. **Search**: filter competitors client-side by name (case-insensitive includes)
    11. List items wrapped in `motion.div` from framer-motion: `initial={{ opacity: 0, y: 8 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.2 }}`

    **AddCompetitorDialog.tsx** — Follow NewProfileDialog.tsx pattern (per D-14):

    1. Props: `{ open: boolean; onOpenChange: (open: boolean) => void; onCreated: () => void }`
    2. Form state: name (required), page_id, page_url, grupo, notas (all optional)
    3. Validation: name required — if empty on submit show "Nome e obrigatorio" error in `var(--cr-red)` at 12px below the field
    4. Submit: `supabase.from('ad_competitors').insert({ name, page_id: page_id || null, page_url: page_url || null, grupo: grupo || null, notas: notas || null })`
    5. On success: toast.success, call onCreated(), close dialog
    6. On error: toast.error
    7. Submit button: primary variant, label "Adicionar Concorrente" (per UI-SPEC copywriting)
    8. Use Dialog component from components/ui/dialog

    **EditCompetitorDialog.tsx** — Follow EditProfileDialog.tsx pattern:

    1. Props: `{ competitor: AdCompetitor | null; open: boolean; onOpenChange: (open: boolean) => void; onUpdated: () => void }`
    2. Pre-populate form from competitor prop using useEffect
    3. Same fields as Add but pre-filled
    4. Submit: `supabase.from('ad_competitors').update({ name, page_id, page_url, grupo, notas }).eq('id', competitor.id)`
    5. Submit button label: "Salvar Alteracoes"
